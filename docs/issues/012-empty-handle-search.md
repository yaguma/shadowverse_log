# handleSearch()の空実装

## 概要

`StatisticsDashboardPage` の `handleSearch` 関数が空実装になっている。useEffectで自動取得しているため不要だが、コードの意図が不明確。

## 優先度

🟡 低

## 該当箇所

| ファイル | 行 |
|----------|-----|
| `apps/web/src/pages/StatisticsDashboardPage.tsx` | 192-195 |

## 現状

```typescript
const handleSearch = () => {
  // useEffectで自動取得するため、ここでは何もしない
  // フィルター変更時に自動的にデータが再取得される
};
```

## 問題点

1. 関数が存在するのに何もしないのは混乱を招く
2. 将来のメンテナンス時に誤解を生む可能性
3. 手動検索ボタンがUIに存在する場合、クリックしても何も起きない

## 改善案

### Option 1: 関数を削除

useEffectで自動取得しているなら、検索ボタン自体を削除するか、関数を削除する。

```typescript
// 検索ボタンを削除し、フィルター変更時の自動取得のみにする
```

### Option 2: 明示的な再取得を実装

```typescript
const handleSearch = () => {
  // 明示的にデータを再取得
  fetchStatistics(currentFilters);
};
```

### Option 3: コメントを改善

どうしても空実装を残す場合は、より詳細なコメントを追加。

```typescript
/**
 * 検索ボタンのハンドラ
 *
 * 注: 現在はuseEffectでフィルター変更を監視して自動取得しているため、
 * この関数は意図的に空実装としています。
 * 将来、手動リフレッシュ機能を追加する場合はここに実装します。
 */
const handleSearch = () => {
  // 意図的な空実装
};
```
